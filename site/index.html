Get started

Pick your flavor, then run three commands

Every flavor is the same OCI image whether you boot it on bare metal or run it under podman. Pick one of the 18 names from the matrix below and substitute it for <flavor> in the commands.

Example using niri-firefox-flatpak:

1 Switch a running bootc system

Atomic upgrade in place. Reboot to apply.

bootc switch ghcr.io/vaskozl/<flavor>:latest

2 Try it in a container

Pull the image and poke around — no commitment.

podman run --rm -it ghcr.io/vaskozl/<flavor>:latest /bin/bash
3. Install to disk

Boot any Linux live USB or an existing system. Pull the flavor image, then run bootc install to-disk against the target device.

podman run --rm --privileged --pid=host -v /dev:/dev -v /:/target \
  ghcr.io/vaskozl/<flavor>:latest \
  bootc install to-disk /dev/sdX

Replace /dev/sdX with your target disk. This wipes the disk.

Atomic & auto-updating

bootc swaps the base image; podman auto-updates containers. No package drift, no yum/apt half-states.

No installer

Run bootc install to-disk from any live USB. No anaconda, no Calamares, no ISO factory.

Immutable root

System files are read-only. User state lives in /var and $HOME. Rollback is one bootc command.

Common questions

Frequently asked

Why "Sheltie Linux"?
Inspired by Puppy Linux flavors. Same spirit (small, friendly, directly usable, atomic), different breed. Sheltie images are bootc OCI containers — pull and run, no installer needed.
How do I update?
You don't. bootc updates the base image; podman auto-updates containers. Reboot to pick up new bootc layers.
Can I customise?
Build your own flavor: fork the repo, add a flavors/<name>.yaml, open a merge request. Same matrix, one extra row.
Where do I report bugs?
File an issue at gitlab.sko.ai/doudous/apkontainers/-/issues.
Is this an image factory?
No. Sheltie ships a fixed matrix of 18 flavors. The site exposes that matrix; it doesn't generate custom images on demand.
